4技术设计

存储

队列文件
/tmp/feedback_gate_queue_{pid}.json
格式
{ "items": [{ "id": 1, "text": "...", "status": "pending" }] }
管理者
Extension 负责写入（用户入队）和读取消费（Agent trigger 时）

Extension 改动

UI
底部面板加 正常/队列 模式切换按钮
队列模式 UI
显示队列列表（序号、内容、状态）+ 删除按钮
输入行为
队列模式下发送 = 追加到队列文件（不写 response）
trigger 处理
收到 trigger 时：队列非空 → 取队首写 response；空 → 弹窗

MCP Server 改动

改动量
无需改动。队列逻辑完全在 Extension 端处理。MCP Server 只管 trigger → 等 response。
关键设计决策：队列管理完全在 Extension 端。Extension 收到 trigger 后检查队列文件，有内容就立即写 response 文件（不弹窗），没有才弹窗。MCP Server 无感知，无需修改。

边界情况

随时追加
Agent 处理过程中可继续往队列添加新指令
删除/重排
队列项支持删除（✕ 按钮），暂不支持拖拽重排
模式切换
从队列模式切回正常模式：队列内容保留，但下次 trigger 会弹窗
Cursor 重启
队列持久化在 /tmp 文件，重启后如果文件还在则恢复
